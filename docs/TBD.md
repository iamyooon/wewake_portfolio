## 기존 예측 CAGR 참고 설계 (과거 보고서 CAGR을 입력으로 주기)

**목적:** 실행마다·날마다 CAGR이 12% vs 17% vs 19%처럼 크게 흔들리는 것을 완화. “지난번(또는 최근 N회) 보고서의 최종 CAGR”를 **참고용**으로 프롬프트에 넣어, AI가 완전히 새로 샘플링하기보다 **앵커(기준점)** 로 활용하게 함.

### 구현 방향 (요약)

1. **과거 보고서에서 CAGR 추출**
   - `report/` 아래 최신 보고서 1개(또는 최근 N개)를 읽어, “최종 전략적 CAGR”, “Base(Grok)”, “Base(Gemini)” 등 정규식 또는 간단 파싱으로 추출.
2. **프롬프트에 주입**
   - Step 1(Grok) 유저 프롬프트 또는 포트폴리오 참고 블록에  
     `**참고(과거 예측):** 이전 보고서 최종 CAGR 16.9% (Grok α 18%, Gemini β 16.8%). 이번 예측 시 참고하되, 시장·데이터가 달라지면 다른 수치를 제시해도 됨.`  
     형태로 추가.
3. **선택 사항**
   - “참고만 하고 무시 가능” vs “이 구간(예: ±1%p) 안에서 조정 권장” 중 하나를 프롬프트 문구로 명시.

### 장점

- 실행 간·일 간 **숫자 변동이 줄어들 가능성**이 있음 (완전 해결은 아님).
- “어제 16.9%였는데 오늘 12%”처럼 **극단적 점프**를 완화할 수 있음.
- 구현이 상대적으로 단순 (파일 1개 읽기 + 문자열 치환).

### 단점 / 주의

- **시장이 확 바뀐 날**에는 과거 CAGR이 오히려 **잘못된 앵커**가 될 수 있음. 그래서 “참고하되, 근거가 있으면 다르게 제시 가능” 문구가 필요.
- 최신 보고서 파일명·형식이 바뀌면 파싱이 깨질 수 있음. (파일명 패턴 고정 또는 최신 1개 선택 규칙 명시.)
- **첫 실행** 또는 report/ 비어 있으면 “참고 없음”으로 두고 기존처럼 동작하도록 분기 필요.

### 결론 (의견)

- **도입 추천.** temperature=0만으로는 web_search·검색 결과 차이에 따른 변동이 남으므로, “과거 CAGR 참고”는 **추가 완화 수단**으로 의미 있음.
- 단, **“참고일 뿐, 근거 있으면 다른 수치 제시 가능”**을 명시해 두어, 급락·급등 같은 구조 변화 시 AI가 과거 숫자에 묶이지 않도록 하는 것이 좋음.

**결정:** **당장은 적용 안 함.** temperature=0 적용 후 4회 테스트에서 최종 CAGR 변동 폭이 6.7%p → 1.2%p로 충분히 완화되어, 이 기능 없이도 사용 가능. **나중에 변동이 다시 커지거나 일 간 안정성이 필요해지면 그때 구현 검토.**
